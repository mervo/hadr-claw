01 · What

A monitoring agent for HADR

It watches USGS, GDACS and ReliefWeb, merges duplicate reports of the same event, and remembers what it has already seen.

Each morning it publishes one page: what's new, what escalated, what it means — assessed by an AI model that can only speak about events the pipeline verified.

02 · Why

Feeds are for machines

The same quake arrives three times under three IDs, buried in entries that will never matter. Someone has to read it all — every morning.

The claw makes that reading unnecessary. And it's built to be trusted: hard caps on the model, a deterministic fallback, so the report always exists. Next: more feeds, deeper escalation tracking.

03 · Who

The morning duty officer

Analysts, response planners, NGO ops — anyone who needs situational awareness before the day's first meeting, without an overnight shift.

It speeds up exactly one decision:
“Does anything need my attention this morning?”
One glance, not thirty minutes of feeds.

04 · Not

Deliberate non-goals

  • Not a pager. Daily report, not real-time alerts.
  • Not a source. Every fact links back to the official feed.
  • No invented events. Unknown IDs are rejected in code.
  • No runaway spend. Turns, tokens, wall clock — capped.
  • No servers. If a run fails, it files the issue itself.
