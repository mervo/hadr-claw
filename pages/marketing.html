What is this?

A monitoring agent for HADR

A small, always-on agent ("claw") for humanitarian assistance and disaster response. It pulls USGS earthquakes, GDACS multi-hazard alerts and UN OCHA's ReliefWeb, merges duplicate reports of the same event, and diffs against what it has already seen.

One page, once a day

The output is a single static HTML situation report: escalated, new, updated and ongoing events, each with an AI-written assessment of what happened, where, how bad, and who is affected. Facts are injected from the feeds — the model cannot invent events.

Mostly files and a loop

No server. GitHub Actions is the heartbeat, git is the memory, GitHub Pages is the channel, and one docker compose run is the whole system — identical in CI, on a VPS, or on a laptop.

Why it does what it does — and where it's headed

Why

Disaster feeds are noisy, redundant and shaped for machines, not for a person deciding at 08:30 whether today needs attention. The same earthquake arrives three times under three identifiers; most entries never matter to a regional watch floor. Someone — or something — has to read it all every morning.

The claw exists to make that reading unnecessary: deterministic code does the filtering and dedup (so it's testable and repeatable), and a capped model does only the judgement — with a fallback so the report always exists, even when the model fails.

Direction

All seven planned tiers are built, through the capped overnight self-improvement loop. From here the trajectory is depth, not sprawl: more hazard feeds behind the same event schema, richer escalation tracking, and more of the report's quality guarded by deterministic checkers rather than prompts.

The roadmap and its known blind spots are public — see ROADMAP.md.

Who it's for — and the decision it speeds up

Audience

A global HADR watch floor and anyone with the morning duty officer's problem: analysts, response planners, NGO ops staff, or a team lead who needs situational awareness before the day's first meeting — without staffing an overnight shift.

The decision

"Does anything need my attention this morning?" — answered in one glance instead of thirty minutes of feed-reading. Escalations surface above the fold, quiet mornings say "no new developments" in one line, and a page that never changes is distinguishable from a dead one by its freshness stamp.

What it deliberately does not do

  • Not real-time alerting. It is a daily morning report, not a pager. For minute-level warnings, use official channels; the claw tells you what the night left behind.
  • Not a primary source. Every event links back to USGS, GDACS or ReliefWeb. The claw curates and assesses; it never replaces official advisories, and it ingests no social media or unverified reports.
  • No invented facts. The model can only assess events the pipeline verified; unknown event ids are rejected in code, and every fact on the page comes from the normalized feed data.
  • No unbounded spend. Turns, tokens and wall clock are hard-capped in code. If the model misbehaves, a deterministic fallback publishes the report anyway — with a banner saying so.
  • No infrastructure to babysit. No database, no queue, no server. If a run fails, it opens an issue that summons an AI investigator and pings Telegram — silence is never ambiguous.
