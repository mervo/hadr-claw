Architecture & workflow

An unattended disaster-monitoring agent: a deterministic feed pipeline, a capped agentic assessment loop, and a daily GitHub Actions heartbeat that publishes a morning situation report — with a fallback that guarantees the report always exists.

The daily run — 00:00 UTC / 08:00 SGT

HeartbeatActions cron wakes the claw Runner checks out the repo (which is the memory) and runs the same container used in dev: docker compose run --rm claw.
Pipeline · hadr/Fetch → normalize → dedup USGS, GDACS and ReliefWeb are pulled, mapped to one event schema, and cross-feed merged (GLIDE ids, then space-time proximity). No LLM here.
MemoryDiff against what's been seen Events compared with state/seen_events.json: new, escalated, updated, unchanged, deleted. A quiet morning skips the model entirely.
Agent · agent/Model assesses the changes A ~100-line loop briefs the model with the diff; it answers via a structured write_dashboard tool call. Unknown event ids are rejected.
ChannelPublish & commit back Checks gate the report; state/ + dashboard.html are committed to main, and the dashboard deploys to GitHub Pages.

Guardrails — enforced in code, not prose

  • Hard caps on turns, total tokens and wall clock; a 10-minute timeout wraps the whole run in CI.
  • Deterministic fallback: if the model fails or a cap trips, a stdlib renderer still writes the report, with a banner.
  • No invented events: facts (magnitude, place, links) are injected from normalized data by uid; unknown uids are rejected.
  • Gate before publish: check_dashboard.py and check_spend.py must pass or nothing deploys.
  • Failure alerting: a failed run opens a heartbeat-failure issue tagging @claude and pings Telegram.
  • Every string from a feed is HTML-escaped; all timestamps are UTC ISO8601, converted only at the feed boundary.

The two layers

  • hadr/ — deterministic pipeline. Fetch, normalize, cross-feed dedup, memory diff, fallback rendering. Fully testable offline against fixtures; no LLM call in this package, ever.
  • agent/ — agentic engine. The morning run's model loop (OpenCode Zen gateway, OpenAI-compatible). Tests replay recorded transcripts via HADR_FAKE_MODEL — CI never needs a key or the network.
  • Noise filter: quakes kept at mag ≥ 4.5 or sig ≥ 600 or any PAGER alert; GDACS Orange/Red always kept; ReliefWeb (human-curated) always kept.

Development workflow

  • One tier = one branch = one PR = one @claude review — seven tiers, each end-to-end demoable (see ROADMAP.md).
  • Canonical runtime is docker-compose; uv run is the fast dev loop. Tests: uv run pytest · lint: uv run ruff check .
  • The repo is the claw's memory: runners are wiped, so the heartbeat commits state back to main. Dev PRs never touch dashboard.html/state/**.
  • Overnight loop (Tier 7): a capped self-improvement loop — claude -p per iteration against goal.md, with pristine-copy checkers and checkpoint-commit reverts.
  • Hard-won fixes land as one file each in docs/solutions/; secrets live only in .env (gitignored) and Actions secrets.
